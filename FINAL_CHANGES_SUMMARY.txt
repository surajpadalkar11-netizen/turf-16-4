## Testing Steps

### Step 1: Update Database
Run in Supabase SQL Editor:
```sql
-- Add new payout fields
ALTER TABLE bookings ADD COLUMN IF NOT EXISTS razorpay_payout_id TEXT;
ALTER TABLE bookings ADD COLUMN IF NOT EXISTS payout_account TEXT;
```

### Step 2: Configure Admin Razorpay
1. Start admin panel: `cd admin && npm run dev`
2. Login as admin
3. Go to Settings
4. Enter your Razorpay Key ID and Secret
5. Save

### Step 3: Ensure Turf Has Razorpay Account
1. Each turf must have razorpay_key_id configured
2. This is the turf owner's Razorpay account
3. Admin will transfer money to this account

### Step 4: Test Payout Flow
1. User adds money to wallet (goes to admin's Razorpay)
2. User books turf with wallet
3. Admin goes to Payouts page
4. Admin sees:
   - Turf name
   - Owner email
   - Owner's Razorpay Key ID (e.g., rzp_live_xxxxx)
   - Amount to pay
5. Admin clicks "Pay Now"
6. Confirmation shows all details
7. Payout processes
8. Money transfers to turf owner's Razorpay account
9. Status changes to "Completed"
10. Transfer ID displayed

## Money Flow

1. **User → Admin Razorpay**
   - User adds ₹500 to wallet
   - Money goes to admin's Razorpay account
   - User's wallet balance = ₹500

2. **User Wallet → Booking**
   - User books turf for ₹500
   - Wallet balance deducted
   - Booking marked as "wallet payment"
   - Payout status = "pending"

3. **Admin Razorpay → Turf Owner Razorpay**
   - Admin processes payout
   - ₹500 transferred from admin's account
   - Goes to turf owner's Razorpay account
   - Payout status = "completed"
   - Transfer ID recorded

## Important Notes

1. **Razorpay Accounts Required**:
   - Admin: 1 Razorpay account (receives all wallet top-ups)
   - Each Turf Owner: 1 Razorpay account (receives payouts)

2. **Razorpay Payouts API**:
   - Currently marked as "manual" transfer
   - To enable automatic transfers, need Razorpay X account
   - Uncomment Razorpay Payouts API code in payoutController.js

3. **Security**:
   - All Razorpay keys encrypted in database
   - Only admin can process payouts
   - Validates before processing
   - Records audit trail

4. **Validation**:
   - Cannot pay if turf owner has no Razorpay account
   - Cannot pay twice
   - Cannot pay if no wallet payment exists
   - Shows warning if Razorpay account missing
